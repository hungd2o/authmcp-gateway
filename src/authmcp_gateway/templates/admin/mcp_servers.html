Pipeline Call Config

                               

                                   
                                   
                                   
Each step must include a command and optional args array.

                               

                               

                                   
                                   
                               

                               

                                   
                                   
                               

                               

                                   
                                   
                                   
Only the first step receives the tool's arguments as JSON on stdin. Each later step receives the previous step's raw stdout as its stdin (plain text passthrough, not re-parsed against this schema) — chain steps like script.py | jq.

                                   
 Each step's stdout/stderr are capped at 256KB and truncated beyond that; a non-zero exit code stops the pipeline and is returned as a tool error.
